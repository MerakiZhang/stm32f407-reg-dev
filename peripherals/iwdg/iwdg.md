# iwdg — 独立看门狗（IWDG）

## 概述

独立看门狗（Independent Watchdog，IWDG）是 STM32F407 的硬件安全机制，用于检测软件跑飞或死锁。它由独立的 LSI 内部 RC 振荡器（约 32 kHz）驱动，与主系统时钟完全无关，即使主时钟停摆 IWDG 依然运行。

若主程序在超时时间内未能喂狗，IWDG 将强制产生系统复位。

---

## 硬件资源

| 资源 | 说明 |
|------|------|
| 时钟源 | LSI 内部 RC，约 32 kHz（精度约 ±20%，误差较大） |
| 基地址 | `0x40003000`（APB1 总线） |
| 复位标志 | `RCC->CSR` bit29 `IWDGRSTF`，可检测上次是否为看门狗复位 |
| 调试暂停 | `DBGMCU->APB1FZ` bit12 `DBG_IWDG_STOP`，置 1 可在调试暂停时冻结计数 |

> **注意**：IWDG 一旦启动，软件无法关闭，复位也不会停止（仅电源断电才停止）。

---

## 寄存器说明

### KR — 键值寄存器（只写）

| 写入值 | 作用 |
|--------|------|
| `0x5555` | 解锁 PR/RLR 寄存器写访问 |
| `0xAAAA` | 喂狗（将 RLR 值重载入计数器） |
| `0xCCCC` | 启动 IWDG |

### PR — 预分频寄存器（bit2:0）

| PR 值 | 分频系数 | LSI=32kHz 时每 tick 时长 |
|-------|----------|--------------------------|
| 0 | /4 | 0.125 ms |
| 1 | /8 | 0.25 ms |
| 2 | /16 | 0.5 ms |
| 3 | /32 | 1 ms |
| 4 | /64 | 2 ms |
| 5 | /128 | 4 ms |
| 6 | /256 | 8 ms |

### RLR — 重载值寄存器（bit11:0，范围 0~4095）

超时时间公式（LSI ≈ 32 kHz）：

```
T(ms) = (RLR + 1) × prescaler / 32
```

### SR — 状态寄存器

| 位 | 名称 | 说明 |
|----|------|------|
| 0 | PVU | 预分频值更新中，等待 LSI 同步 |
| 1 | RVU | 重载值更新中，等待 LSI 同步 |

写 PR/RLR 后，SR 对应位置 1，同步完成后自动清 0。

---

## 超时范围

| PR 值 | 最小超时（RLR=0） | 最大超时（RLR=4095） |
|-------|------------------|---------------------|
| 0 (/4)   | 0.125 ms | 512 ms |
| 1 (/8)   | 0.25 ms  | 1024 ms |
| 2 (/16)  | 0.5 ms   | 2048 ms |
| 3 (/32)  | 1 ms     | 4096 ms ≈ 4 s |
| 4 (/64)  | 2 ms     | 8192 ms ≈ 8 s |
| 5 (/128) | 4 ms     | 16384 ms ≈ 16 s |
| 6 (/256) | 8 ms     | 32768 ms ≈ 32 s |

---

## API

```c
void iwdg_init(uint32_t timeout_ms);
```
初始化并启动 IWDG。自动根据 `timeout_ms` 选择最小预分频（使 RLR 不超过 4095）。  
有效范围约 `1 ~ 32768 ms`（PR=6/÷256，RLR=4095：4096×256/32000×1000=32768 ms），超出范围自动截断为最大超时。

```c
void iwdg_feed(void);
```
喂狗，重载计数器。必须在主循环中以小于 `timeout_ms` 的间隔定期调用。

---

## 使用示例

```c
#include "iwdg.h"

int main(void) {
    sys_clock_init();
    delay_init();

    iwdg_init(1000);   /* 超时 1000ms，超时后自动复位 */

    while (1) {
        iwdg_feed();   /* 在每次主循环开始时喂狗 */

        /* 正常业务逻辑 */
        delay_ms(200);
    }
}
```

## 检测看门狗复位

上电时可读取 `RCC->CSR` 的 `IWDGRSTF` 位判断上次是否由 IWDG 触发复位：

```c
if (RCC->CSR & (1U << 29)) {
    /* 上次是 IWDG 复位 */
    RCC->CSR |= (1U << 24);   /* 写 RMVF 位，清除所有复位标志 */
}
```

---

## 调试技巧

调试时若单步暂停会触发 IWDG 超时，可配置调试暂停时冻结 IWDG：

```c
/* 在 iwdg_init 之前调用 */
DBGMCU->APB1FZ |= (1U << 12);   /* DBG_IWDG_STOP: 调试暂停时冻结 IWDG 计数 */
```

---

## 注意事项

1. **LSI 精度低**：LSI 误差约 ±20%，实际超时可能偏短，喂狗间隔建议为超时时间的 1/2 以下。
2. **不可关闭**：IWDG 一旦启动，不可通过软件停止，也不受软件复位影响。
3. **与 `delay_ms` 兼容**：`delay_ms` 是阻塞延时，期间无法喂狗，需确保单次延时小于超时时间。
4. **与 WWDG 的区别**：IWDG 适合防死锁（超时复位），无上限检测；WWDG 有窗口约束，适合检测时序异常。

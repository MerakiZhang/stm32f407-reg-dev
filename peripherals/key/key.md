# 按键驱动说明

## 硬件连接

| 标识   | 引脚 | 连接方式             | 有效电平 |
|--------|------|----------------------|----------|
| KEY_UP | PA0  | 一端接 VCC 3.3V      | 高电平有效（按下为高） |
| KEY0   | PE4  | 一端接 GND           | 低电平有效（按下为低） |
| KEY1   | PE3  | 一端接 GND           | 低电平有效（按下为低） |
| KEY2   | PE2  | 一端接 GND           | 低电平有效（按下为低） |

- KEY_UP 按下时 PA0 读到高电平，松开时由内部下拉电阻拉低，读到低电平
- KEY0/KEY1/KEY2 按下时对应引脚读到低电平，松开时由内部上拉电阻拉高，读到高电平

---

## 寄存器配置说明

### 第一步：使能 GPIOA 和 GPIOE 时钟

寄存器：**RCC_AHB1ENR**（RCC AHB1 外设时钟使能寄存器）

| 位   | 名称    | 说明 |
|------|---------|------|
| 位 0 | GPIOAEN | 置 1 使能 GPIOA 时钟 |
| 位 4 | GPIOEEN | 置 1 使能 GPIOE 时钟 |

```c
RCC->AHB1ENR |= (1U << 0);   // 使能 GPIOA
RCC->AHB1ENR |= (1U << 4);   // 使能 GPIOE
```

---

### 第二步：配置引脚模式（MODER）

寄存器：**GPIOA_MODER** / **GPIOE_MODER**（GPIO 端口模式寄存器）

每个引脚占 2 位，引脚 n 对应位 [2n+1 : 2n]：

| 位值 | 模式 |
|------|------|
| 00   | 输入模式 |
| 01   | 通用输出模式 |
| 10   | 复用功能模式 |
| 11   | 模拟模式 |

按键引脚均设置为输入模式（00）。

PA0 对应位 [1:0]，设置为 **00**：
```c
GPIOA->MODER &= ~(3U << (0 * 2));
```

PE4 对应位 [9:8]，PE3 对应位 [7:6]，PE2 对应位 [5:4]，均设置为 **00**：
```c
GPIOE->MODER &= ~((3U << (4 * 2)) | (3U << (3 * 2)) | (3U << (2 * 2)));
```

---

### 第三步：配置上下拉（PUPDR）

寄存器：**GPIOA_PUPDR** / **GPIOE_PUPDR**（GPIO 端口上下拉寄存器）

每个引脚占 2 位，引脚 n 对应位 [2n+1 : 2n]：

| 位值 | 配置 |
|------|------|
| 00   | 无上下拉 |
| 01   | 上拉 |
| 10   | 下拉 |
| 11   | 保留 |

**KEY_UP（PA0）：内部下拉**

KEY_UP 按下时引脚接 VCC（高电平），松开时引脚悬空，需要内部下拉确保读到低电平。PA0 位 [1:0] 设置为 **10**：
```c
GPIOA->PUPDR &= ~(3U << (0 * 2));
GPIOA->PUPDR |=  (2U << (0 * 2));
```

**KEY0/KEY1/KEY2（PE4/PE3/PE2）：内部上拉**

KEY0~KEY2 按下时引脚接 GND（低电平），松开时引脚悬空，需要内部上拉确保读到高电平。PE4/PE3/PE2 对应位设置为 **01**：
```c
GPIOE->PUPDR &= ~((3U << (4 * 2)) | (3U << (3 * 2)) | (3U << (2 * 2)));
GPIOE->PUPDR |=  ((1U << (4 * 2)) | (1U << (3 * 2)) | (1U << (2 * 2)));
```

---

### 第四步：读取引脚电平（IDR）

寄存器：**GPIOA_IDR** / **GPIOE_IDR**（GPIO 端口输入数据寄存器）

IDR 是只读寄存器，位 n 反映引脚 n 当前电平：

| 位值 | 状态 |
|------|------|
| 0    | 引脚当前为低电平 |
| 1    | 引脚当前为高电平 |

读取示例：
```c
// 读取 KEY_UP（PA0），为 1 表示按下
(GPIOA->IDR & (1U << 0))

// 读取 KEY0（PE4），为 0 表示按下
(GPIOE->IDR & (1U << 4))
```

---

## 按键消抖

按键按下时机械触点存在抖动，通常持续 5~10ms，直接读取会误判多次触发。消抖方式为：检测到电平变化后延时 10ms，再次读取确认电平一致，才认为按键有效按下。

```
检测到按键变化 → 延时 10ms → 再次读取 → 确认后返回按键值
```

## 函数说明

### key_init

初始化函数。配置 PA0 为输入模式 + 内部下拉，配置 PE4/PE3/PE2 为输入模式 + 内部上拉。

### key_scan(uint8_t mode)

按键扫描函数。扫描四个按键，返回当前按下的按键编号，无按键按下返回 0。

参数 `mode`：
- `0`：单次触发模式，按住不松开只返回一次
- `1`：连续触发模式，按住持续返回按键值

返回值：

| 返回值      | 数值 | 含义 |
|-------------|------|------|
| KEY_UP_VAL  | 4    | KEY_UP 按下（PA0 高电平） |
| KEY0_VAL    | 3    | KEY0 按下（PE4 低电平） |
| KEY1_VAL    | 2    | KEY1 按下（PE3 低电平） |
| KEY2_VAL    | 1    | KEY2 按下（PE2 低电平） |
| 0           | 0    | 无按键按下 |

扫描优先级：KEY_UP > KEY0 > KEY1 > KEY2。
